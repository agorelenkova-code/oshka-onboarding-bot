## 2. Архитектура и потоки данных

```
                 [ Google Sheets «Лист1» — прогресс ]
                          ▲            ▲
        Apps Script       │            │  fetch (no-cors, webtoken)
        (doGet/doPost)    │            │
   ┌──────────────────────┘            └───────────────────────┐
   │ secret (сервер)                        webtoken (браузер) │
[ Бот на Render ]                                              │
   │  ▲                                                        │
   │  │ HTTP /progress (initData HMAC)                         │
   ▼  │                                                        │
[ Курс — Mini App ]  ── Telegram-пользователь                  │
[ Курс — сайт ]      ── пользователь без Telegram ────────────┘  (ключ = web:<email>)
```

**Сценарии:**
- **Регистрация.** На старте курса человек указывает роль, имя, email, класс(ы), тариф. Из Telegram — уходит боту (`status: register`) → в таблицу (ФИО, класс). С сайта — пишется напрямую в таблицу (`source: web`).
- **Прогресс.** Каждый пройденный блок → `localStorage['oshka_done']` + запись в таблицу (`status: done` → ✅ в колонке «задание N»).
- **Вопросы.** Кнопка «Остались вопросы?» на блоке → `status: question` → пишется в колонку «вопросы» таблицы и падает в **чат кураторов** (Telegram). Для Telegram-пользователей уведомляет сам бот (в `handle_progress`); для **веб-пользователей** (без Telegram) Apps Script дёргает эндпоинт бота `POST /web-question` (под общим секретом), и бот пересылает вопрос кураторам с ФИО+классом из строки таблицы.
- **Напоминания.** Раз в сутки (по умолчанию 10:00 МСК) бот читает таблицу. Если этап **не пройден** к **следующему дню** после открытия — срабатывает **одно** напоминание к первому непройденному видимому блоку (`REMIND_AFTER_DAYS=1`, ключ `S{n}:r`, повторов нет). Этап пройден = все видимые блоки сделаны ЛИБО начат более поздний этап. Доставка по типу `user_id`:
  - **Telegram** (числовой id) — бот сам шлёт дружеское напоминание;
  - **веб-пользователи** (`web:<email>`, без Telegram) — бот НЕ шлёт письма сам, а собирает их в **ежедневную сводку «кому напомнить»** и отправляет в **чат кураторов** (`CURATORS_CHAT_ID`). В сводке: имя, почта, класс/тариф, какой этап не пройден, prefill-ссылка `?e=&n=&k=&t=&r=` + готовый шаблон письма. **Куратор пишет родителю со своей почты.** Отметка «напомнили» ставится только если сводка ушла.
  - Telegram-ограничение: бот пишет лишь тем, кто нажал Start.

**Ключ пользователя в таблице:** Telegram — `user_id` (число), веб — `web:<email>`.

**Персонализация (видимость блока зависит от роли · тарифа · класса).** Матрица `VIS` продублирована в `index.html` и `tg-onboarding.js`. Тарифы: `sz` (с зачислением), `chsz` (частичное), `bz` (без зачисления), `veb` (вебинар), `oa` (организация аттестации), `dostup` (без учителя / доступ к материалам). Логика `isVisible(task, profile)`:
- тариф выбран и блока нет в списке разрешённых → скрыть;
- роль не в списке разрешённых → скрыть;
- у блока задан список классов и ни один класс профиля в него не входит → скрыть.

Родитель с **несколькими детьми** вводит несколько классов (напр. «7, 11») → показывается **объединение** видимого контента. Тариф `dostup` видит только 4 базовых блока (1, 2, 4, 21). Блок «Документы» (9) — только родителю. Блок «ИС + ЕГЭ/ОГЭ» (19) — только тарифы `sz`/`veb` и класс 9 или 11.

**Наглядная карта блоков и сравнение маршрутов:** интерактивная страница-навигатор (см. §11).

## 4. Ключевые файлы

**Бот (`onboarding-bot/`):**
- `bot.py` — команды `/start`, `/reset`, `/id`; HTTP-роуты `/progress` (POST, приём прогресса), `/cron/reminders` (GET, ручной триггер рассылки, под секретом), `/` (health); тексты приветствия/тизеров; логика напоминаний (`run_reminders`, `send_reminder`, `reminder_loop`); расписание `LESSON_*`.
- `storage.py` — обмен с Apps Script: `record`, `register`, `fetch_users`, `get_user`, `mark_reminder`; есть локальный fallback `progress.json` (эфемерный, для отладки).
- `apps-script.gs` — код для Google-таблицы (в репозитории — с **заглушкой** секрета `SECRET`). Содержит `doPost` (запись; авторизация по `secret` для бота или `webtoken` для браузера), `doGet` (чтение прогресса по `secret`; плюс сервисные экшены для чтения листов/картинок из докmethods).
- `render.yaml`, `Procfile`, `requirements.txt`.
- `.env` — секреты, **локально, в git не идёт** (в `.gitignore`).
- `pull_course_sheet.sh` + `course_sheet_baseline.json` — синхронизация контента из «Лист7» (см. §9).

**Курс (`onboarding-course/`):**
- `index.html` — стартовый экран (роль/имя/email/классы/тариф) + карта блоков + матрица `VIS`.
- `task-*.html` — блоки курса.
- `finish.html` — финальный экран.
- `tg-onboarding.js` — вся связка: регистрация, прогресс, вопросы, автопереход, тизеры-навигация, дубликат `VIS`, ветка веб-прогресса (`webPost` → `fetch(mode:"no-cors")`), метка версии. Подключён на каждой странице.
- `config.js` — адреса API: `TG_ONBOARDING_API` (бот), `TG_PROGRESS_API` (Apps Script), `TG_WEB_TOKEN` (**write-only** токен веб-записи).
- `_headers` — `no-store` для html/js (чтобы Telegram/браузер не кэшировали старое).

## 7. Безопасность — сводка

- ✅ Весь трафик HTTPS; Telegram `initData` проверяется по HMAC (`BOT_TOKEN`); `.env` не в git.
- ⚠️ **Критично при утечке** `BOT_TOKEN`/`APPS_SCRIPT_SECRET` → полный контроль бота / чтение всех ПДн. 2FA на Google/Render/GitHub; при подозрении — ротация токенов.
- ⚠️ `TG_WEB_TOKEN` публичный (в `config.js`) → возможен спам-дозапись строк (не чтение). Митигейт: валидация на стороне Apps Script.
- ⚠️ ПДн несовершеннолетних → согласие + ограничение доступа (см. §6).

Полный разбор рисков по severity и приоритеты — в `PLAN_NEXT.md`.

## 8. Эксплуатация (runbook)

**Проверка здоровья бота:** `curl -s -o /dev/null -w "%{http_code}" https://oshka-onboarding-bot.onrender.com/` → ожидаем `200`.

**Деплой бота.** Репозиторий подключён к Render «по ссылке», поэтому авто-деплой по push может быть **выключен** — деплой запускается хуком или вручную:
```bash
git push origin main
curl -s "$RENDER_DEPLOY_HOOK"          # хук в .env; либо Render → сервис → Manual Deploy → Deploy latest commit
```

**Деплой курса** (прод через `--prod` даёт Forbidden — сайт привязан к Git; обход: draft + restore):
```bash
netlify deploy --dir onboarding-course --site <SITE_ID> --json   # взять deploy_id
netlify api restoreSiteDeploy --data '{"site_id":"<SITE_ID>","deploy_id":"<id>"}'
```
После правок курса — поднять `?v=N` у `config.js`/`tg-onboarding.js` и `VERSION` в `tg-onboarding.js` (сброс кэша Telegram/браузера).

**Изменение Apps Script** (при смене схемы таблицы или логики): вставить обновлённый `apps-script.gs` (**с реальным** `SECRET`) → Сохранить → Развернуть → Управление развёртываниями → ✏️ → Новая версия. При смене `/exec` URL — обновить `APPS_SCRIPT_URL` у бота и `TG_PROGRESS_API` в `config.js`.

**Ручной запуск рассылки:** `GET /cron/reminders?secret=<APPS_SCRIPT_SECRET>`.

**Чтение прогресса (для отладки):** `GET <APPS_SCRIPT_URL>?action=users&secret=<SECRET>` → JSON пользователей.

## 11. Известные ограничения и бэклог

**Ограничения:**
- Прогресс хранится в `localStorage` устройства (дублируется в таблицу). При смене устройства локальный прогресс теряется (кросс-девайс не реализован).
- Напоминания бота пока **не фильтруются матрицей видимости** — бот знает только класс, не тариф/роль; теоретически может позвать в скрытый для профиля блок. → передавать в бот тариф/роль при регистрации.
- Синхронизация контента — полуавтоматическая (§9).

**Бэклог (приоритеты — в `PLAN_NEXT.md`):**
- **P1:** тариф/роль/email в бот при регистрации + фильтр напоминаний по матрице; **email-приветствие и email-напоминания для веб-пользователей** (SMTP Яндекс / SendGrid / Mailgun + планировщик на базе `reminder_loop`, брать email из «Лист1»); согласие на обработку ПДн + ограничение доступа к таблице.
- **P2:** платный Render; аналитика прохождения (сколько дошли, отвал по блокам); кросс-девайс прогресс (подтягивать из таблицы по email/tg-id); эндпоинт удаления строк в Apps Script.

**Навигатор по курсу** (карта блоков + интерактивное сравнение маршрутов по роли/тарифу/классу): страница-обзор — приватный артефакт, ссылку выдаёт владелец.

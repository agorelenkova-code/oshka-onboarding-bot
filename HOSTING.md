## Если захочется уйти от Railway

Код к Railway не привязан (обычный Python + `Procfile`), поэтому переезд простой:
- **Свой VPS** — самый дешёвый/надёжный: `pip install -r requirements.txt` + `python bot.py`
  под systemd. Полностью бесплатно, если сервер уже есть.
- **Render (free)** — честно бесплатно, НО засыпает после 15 мин простоя и убивает
  polling. Чтобы там жить, нужно либо перевести бота в **webhook-режим**, либо держать
  внешний пинг (cron-job.org) на `/` каждые ~10 минут. Менее надёжно.
- **Fly.io / Koyeb** — есть бесплатные тарифы без засыпания, но требуют карту/настройку.

## Что важно при переезде
Перенести переменные окружения (см. `.env.example`): `BOT_TOKEN`, `COURSE_URL`,
`ADMIN_CHAT_ID`, `APPS_SCRIPT_URL`, `APPS_SCRIPT_SECRET`. И обновить адрес API бота в
курсе — `config.js` (`window.TG_ONBOARDING_API`) + передеплой курса на Netlify.

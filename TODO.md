# Что доработать в рабочей версии онбординга

Список того, что осталось/нужно поменять. Текущая версия (бот + курс Mini App + таблица)
рабочая, это улучшения и открытые направления.

## 1. Хостинг — уйти с бесплатного Render
- Сейчас бот на **Render free**, который **засыпает через 15 мин простоя**.
- Временно держим живым через авто-пинг (GitHub Actions, `.github/workflows/keepalive.yml`,
  раз в ~10 мин). Минус Actions: расписание может **задерживаться**, изредка >15 мин — тогда
  возможен короткий сон и задержка первого ответа.
- **Надёжное решение:** платный **Render Starter (~$7/мес)** — засыпание уходит совсем,
  пинг можно удалить. Или перенести на свой VPS (бесплатно, если сервер есть). См. `HOSTING.md`.
- Альтернатива пингу понадёжнее без оплаты: внешний монитор **UptimeRobot / cron-job.org**
  на адрес бота каждые 5 мин.

## 2. LMS — встроить онбординг в собственную платформу (ЗАПАРКОВАНО)
- Цель: онбординг живёт внутри вашей LMS (личный кабинет), прогресс виден на платформе и
  уходит в таблицу; бот ведёт туда, а не в отдельный Mini App.
- Чтобы продолжить, нужны **вводные по LMS**:
  - есть ли доступ к её коду / кто её разрабатывает и ведёт;
  - есть ли у неё API или возможность добавлять страницы/встраивать;
  - есть ли вход и личные кабинеты учеников (к чему привязывать прогресс).

## 3. Кнопки «получилось / не получилось» после КАЖДОГО шага во всех уроках
- Сейчас пошаговые кнопки есть только в уроках с шагами: **task-2 (Платформа), task-3 (Zoom)**.
- Одно-кнопочные уроки (1, 4, 5, 6, 7, 8, 9, 13, 14, 15, 16) надо **разбить на шаги** вручную,
  чтобы у каждого шага была своя пара кнопок. Это контентная работа по каждому уроку.

## 4. Мелочи
- Удалить тестовые строки в Google-таблице (`999000`, `999`), если ещё не удалены.
- `apps-script.gs` в публичном репо лежит с заглушкой секрета (`ВПИШИ_СЕКРЕТ_КАК_В_БОТЕ`) —
  настоящий секрет только в Google-скрипте и в переменных Render. При повторной вставке скрипта
  не забыть подставить реальный `APPS_SCRIPT_SECRET`.
- Финальный сквозной тест в Telegram (выполнил/вопрос → таблица + уведомление) — прогнать на
  «живой» версии и убедиться, что всё долетает.
